1. 실물 플래너 → 웹앱 전환 설계
실물 플래너의 모든 활동을 인터랙티브한 웹앱 경험으로 전환하는 설계입니다. 단순히 PDF를 화면에 보여주는 것이 아니라, 웹앱만의 장점(자동 계산, AI 지원, 실시간 시각화, 데이터 연결)을 최대한 살린 설계입니다.
1-1. PART 1: 나의 현실 마주하기 (5개 활동)
활동 1: 기술과 경험 목록 만들기
실물 플래너 형태: 세 영역(업무경험/개인적 기술/학습 경험)으로 나뉜 빈 페이지에 자유롭게 기록. 마지막에 세 영역을 조합하여 가치를 발견하는 질문 페이지.
웹앱 UI 설계: 
	•	3개 탭(업무경험 / 개인적 기술 / 학습 경험)으로 구분된 입력 화면
	•	각 탭에서 + 버튼을 누르면 항목 추가 카드가 생성됨. 카드에는 '기술/경험명', '설명', '숙련도(1~5 별점)'를 입력
	•	드래그앤드롭으로 카드 순서 변경 및 중요도 정렬 가능
	•	입력한 항목들이 태그 클라우드(워드 클라우드) 형태로 실시간 시각화
웹앱 추가 가치: 
	•	AI 자동 제안: 사용자가 '엑셀'을 입력하면 AI가 '데이터 분석', '피벗테이블', 'VBA 매크로' 등 관련 세부 기술을 자동 제안
	•	조합 엔진: 세 영역의 기술을 AI가 자동 조합하여 '당신이 제공할 수 있는 가치' 3~5개를 추천. 예: 엑셀 + 식물 키우기 → '반려 식물 성장 일지 엑셀 템플릿 제작'
	•	PART 3 자동 연결: 여기서 입력한 기술 데이터가 PART 3의 MVP 구상 시 자동으로 참조됨

활동 2: 나의 자원 지도 그리기
실물 플래너 형태: 6개 자원(금전적/시간적/기술·지식/경험/인적/물리적)을 표에 기록하고 1~5점 매김. 다음 페이지에 방사형 차트를 손으로 그림.
웹앱 UI 설계: 
	•	6개 자원 각각에 대해 슬라이더(1~5점)로 점수를 입력하고, 옆에 텍스트 필드로 구체적 내용 기록
	•	슬라이더를 움직이면 화면 상단의 방사형 차트(레이더 차트)가 실시간으로 업데이트
	•	차트는 인터랙티브: 각 꼭짓점을 클릭하면 해당 자원의 상세 정보 팝업
웹앱 추가 가치: 
	•	AI 인사이트: '당신의 가장 강력한 자원은 경험(5점)입니다. 이를 활용한 아이디어: 경험 기반 멘토링, 사례 연구 콘텐츠 제작...'
	•	취약 자원 보완 가이드: 점수가 낮은 자원에 대해 '이 자원을 높이기 위한 추천 행동' 제시
	•	시간 경과 비교: 3개월 후 다시 평가하면 이전 차트와 오버레이하여 성장 추이 확인

활동 3: 내 시간의 사용법 관찰하기
실물 플래너 형태: 월~일 시간대별 표에 수기로 활동 기록. 이후 생산(초록)/소비(노란)/필수(회색) 시간으로 분류하고 총 시간 계산.
웹앱 UI 설계: 
	•	타임 블록 UI: 일주일 캘린더 형태. 시간 블록을 드래그하여 생성하고, 활동 이름을 입력
	•	색상 자동 분류: 활동을 입력하면 AI가 자동으로 생산(초록)/소비(노란)/필수(회색) 색상 부여. 사용자가 수동 변경도 가능
	•	비율 자동 계산: 화면 하단에 도넛 차트로 생산/소비/필수 시간 비율 실시간 표시
웹앱 추가 가치: 
	•	패턴 분석: '화요일과 목요일 저녁에 소비 시간이 집중됩니다. 이 시간을 생산 시간으로 전환하면 주 6시간을 확보할 수 있습니다'
	•	골든 타임 발견: AI가 사용자의 생산적 시간대 패턴을 분석하여 '당신의 골든 타임은 오전 6~8시입니다' 같은 인사이트 제공
	•	템플릿 제공: '직장인', '프리랜서', '학생' 등 유형별 기본 타임로그 템플릿으로 빠른 시작 가능

활동 4: 돈의 흐름 파악하기
실물 플래너 형태: 한 달간 지출을 표(날짜/항목/구분/금액/만족도)로 기록. 이후 패턴 분석 질문에 답변.
웹앱 UI 설계: 
	•	지출 입력 카드: 날짜 선택 + 항목명 입력 + 카테고리 드롭다운(식비/교통/쇼핑/자기계발/문화 등) + 금액 입력 + 만족도(상/중/하) 선택
	•	지출 대시보드: 카테고리별 원형 차트 + 만족도별 막대 차트 + 일별 지출 추이 라인 차트가 실시간 업데이트
	•	만족도 히트맵: 각 지출 항목을 만족도에 따라 초록(상)/노란(중)/빨간(하)으로 색상 표시
웹앱 추가 가치: 
	•	AI 패턴 분석: '만족도가 하인 지출의 78%가 배달앱입니다. 월 약 12만 원을 절약할 수 있습니다'
	•	나만의 일 자금 시뮬레이션: '만족도 하인 지출을 줄이면 월 15만 원, 연간 180만 원의 초기 자금을 마련할 수 있습니다'
	•	CSV 가져오기: 카드사 지출 내역 CSV를 업로드하면 자동으로 분류 및 입력

활동 5: 나의 현재 상태 정의하기
실물 플래너 형태: 직업/소속, 역할/책임, 제약 조건, 고민/스트레스, 기회를 각 빈 칸에 서술. 이후 '제약 속 기회 찾기' 질문에 답변.
웹앱 UI 설계: 
	•	5개 카드 레이아웃: 직업/소속 | 역할/책임 | 제약 조건 | 고민/스트레스 | 기회. 각 카드를 클릭하면 텍스트 입력 영역 확장
	•	시각적 밸런스 맵: 입력된 내용을 기반으로 '제약 vs 기회' 저울 애니메이션 표시
	•	AI 대화형 인터페이스: '당신의 제약 조건인 잦은 야근은, 같은 처지의 직장인들이 겪는 문제를 깊이 이해한다는 뜻이기도 합니다. 이것이 어떻게 기회가 될 수 있을까요?' 같은 코칭 질문
웹앱 추가 가치: 
	•	SWOT 자동 생성: 입력된 데이터를 기반으로 간략한 개인 SWOT 분석 차트 자동 생성
	•	비슷한 상황의 성공 사례: AI가 사용자의 현재 상태와 비슷한 조건에서 성공한 사례를 참고용으로 제시

PART 1 회고 (Reflection)
실물 플래너 형태: 5개 항목(예상 밖의 발견/건너뛴 활동/가장 작은 성공/가장 큰 배움/다음 파트 집중할 것)에 자유롭게 기록.
웹앱 UI 설계: 
	•	가이드형 회고 화면: 5개 질문이 카드 형태로 하나씩 표시. 한 질문에 답하면 다음 질문으로 부드럽게 전환
	•	완료 축하 애니메이션과 함께 PART 1 종합 요약 카드 표시
	•	AI가 PART 1 전체 활동 내용을 종합 분석한 인사이트 제공: '당신의 가장 큰 자산은 ○○이고, 가장 보완이 필요한 부분은 ○○입니다'

1-2. PART 2: 나만의 '왜' 발견하기 (5개 활동)
활동 1: 경험 마인드맵 그리기
실물 플래너 형태: 큰 빈 페이지(펼침면) 가운데에 이름을 적고, 학창시절/직장/여행/취미/인간관계 등의 가지를 자유롭게 그림.
웹앱 UI 설계: 
	•	인터랙티브 마인드맵 캔버스: 중앙에 사용자 이름이 자동 배치된 노드. 더블클릭으로 새 가지(노드) 추가
	•	드래그앤드롭으로 노드 위치 이동, 노드 간 선 연결, 색상/크기 변경 가능
	•	기본 가지 템플릿 제공: '학창시절', '직장', '여행', '취미', '인간관계' 등 클릭 한 번으로 가지 추가
	•	확대/축소(줌), 전체 맵 보기, 특정 가지 포커스 기능
웹앱 추가 가치: 
	•	AI 패턴 분석: 완성된 마인드맵을 AI가 분석하여 '반복적으로 나타나는 키워드: 사람, 소통, 연결 → 당신은 사람 간의 연결에서 에너지를 얻는 타입일 수 있습니다'
	•	연관 발견: 서로 다른 가지에 있는 유사한 경험을 AI가 자동으로 연결선으로 표시하여 숨겨진 패턴 발견
	•	드림 브레인 연동: 드림 브레인에서 자주 기록한 관심사가 마인드맵에 '추천 가지'로 자동 제안

활동 2: 실패 이력서 작성하기
실물 플래너 형태: 시기 | 실패 경험 | 얻은 교훈 형태의 표를 수기로 작성. 이후 '실패가 가르쳐준 것들' 질문에 답변.
웹앱 UI 설계: 
	•	타임라인 UI: 세로 타임라인에 실패 경험 카드를 시간순으로 추가. 각 카드에 시기(년도 선택), 실패 경험(텍스트), 얻은 교훈(텍스트) 입력
	•	감정 태그: 각 실패에 감정 태그 추가 가능 (좌절/분노/아쉬움/성장 등)
	•	교훈 하이라이트: 입력한 교훈들이 별도 영역에 카드로 모여 '나의 교훈 모음집' 형성
웹앱 추가 가치: 
	•	AI 교훈 연결: '2018년의 실패에서 배운 역할 분담의 중요성이 2022년의 성공에 영향을 준 것 같습니다' 같은 패턴 발견
	•	실패→강점 변환: AI가 실패 경험을 분석하여 '이 실패들을 통해 당신이 쌓은 핵심 강점: 회복 탄력성, 신중한 판단력' 같은 인사이트
	•	PART 2 활동 3(강점 재정의)과 자동 연결: 여기서 발견된 교훈이 강점 재정의 활동에서 참고 자료로 제공

활동 3: 강점과 약점 재정의하기
실물 플래너 형태: 강점/약점을 나열하는 표 + 약점을 새로운 관점으로 재해석하는 칸. 이후 '약점을 사랑하는 법' 질문.
웹앱 UI 설계: 
	•	3열 카드 레이아웃: 왼쪽(강점 카드 리스트) | 가운데(약점 카드 리스트) | 오른쪽(재정의 카드 리스트)
	•	약점 카드를 클릭하면 오른쪽에 '재정의' 입력 필드가 열리며, 화살표 애니메이션으로 약점→강점 전환을 시각적으로 표현
	•	토글 뷰: '원래 관점'과 '새로운 관점' 사이를 토글하여 비교
웹앱 추가 가치: 
	•	AI 재정의 코칭: 약점을 입력하면 AI가 재정의 힌트를 제공. 예: '낯을 많이 가린다' → AI: '조용히 깊이 생각하고 분석하는 데 강한 성향일 수 있습니다. 1:1 심층 상담이나 서면 컨설팅에 큰 강점이 됩니다'
	•	실패 이력서 연동: 실패 이력서에서 발견된 교훈이 약점 재정의의 참고 자료로 자동 표시

활동 4: 30분 시장성 엿보기
실물 플래너 형태: 유튜브(10분)/서점(10분)/커뮤니티(10분) 3단계 검색 결과를 각각 빈 페이지에 기록. 이후 분석 질문.
웹앱 UI 설계: 
	•	3단계 탭 + 타이머: 각 단계(유튜브/서점/커뮤니티)별로 10분 카운트다운 타이머가 작동. 검색 결과를 메모 카드 형태로 추가
	•	구조화된 입력: 각 검색 결과 카드에 '발견한 것', '사람들의 반응', '놓치고 있는 부분' 필드
	•	종합 분석 보드: 3단계 결과를 한 화면에 모아 보여주는 대시보드
웹앱 추가 가치: 
	•	AI 시장 분석: 입력된 관찰 내용을 AI가 종합하여 '시장 기회 요약', '경쟁 상황', '차별화 포인트' 분석 리포트 자동 생성
	•	PART 3 연결: 시장성 분석 결과가 PART 3의 가설-검증 보드에서 자동 참조

활동 5: Why-What 브릿지 & 아이디어 비틀기
실물 플래너 형태: Why 정의 → What 아이디어 3개 → 각 아이디어를 빼기/더하기/결합/반대로 비틀기 → 최종 아이디어 선택. 여러 페이지에 걸친 워크시트.
웹앱 UI 설계: 
	•	Step-by-Step 위자드: 5단계로 구성 (Why 정의 → 아이디어 3개 브레인스토밍 → 비틀기 매트릭스 → 평가 → 최종 선택)
	•	비틀기 매트릭스: 4x3 그리드(빼기/더하기/결합/반대로 x 3개 아이디어). 각 셀을 클릭하면 입력 필드가 열림
	•	최종 선택 카드: 선택한 아이디어를 강조 표시하고, 선택 이유와 기대 가치를 기록
웹앱 추가 가치: 
	•	AI 브레인스토밍 파트너: '당신의 Why와 기술을 조합하면 이런 아이디어는 어떨까요?' 같은 추가 아이디어 제안
	•	아이디어 비교 점수판: 각 아이디어를 실현 가능성/시장성/열정도 3축으로 자동 점수화하여 레이더 차트로 비교
	•	PART 2의 모든 데이터 종합: 마인드맵의 키워드, 실패 이력서의 교훈, 강점/약점, 시장성 분석 결과가 이 활동의 참고 패널에 요약 표시

1-3. PART 3: 0원으로 아이디어를 실험하기 (4개 활동)

	•	투표형 선택: 생성된 제안 중 마음에 드는 것에 하트를 눌러 최종 후보 선정
웹앱 추가 가치:

	•	PART 2 자동 참조: PART 2에서 정의한 Why, 발견한 아이디어가 자동으로 빈칸 추천값으로 제안

웹앱 UI 설계:

	•	실행 계획 체크리스트: 10단계가 체크리스트로 변환되어 하나씩 완료 체크 가능
	•	마일스톤 타임라인: 10단계를 시간축에 배치하여 진행도 시각화
웹앱 추가 가치:

	•	PART 1 자동 연결: PART 1에서 파악한 자원(시간, 기술, 인맥)이 MVP 구상 화면 옆에 참고 패널로 표시

활동 4: 나의 가치 사다리 만들기
실물 플래너 형태: 4단계 사다리(미끼 상품→저가→중가→고가) 각각에 상품 형태/가격/고객 가치를 기록하는 표. 이후 점검 질문.
웹앱 UI 설계: 
	•	시각적 사다리 UI: 화면에 실제 사다리 모양의 4단계가 아래에서 위로 표시. 각 단을 클릭하면 상품명/가격/가치 입력 패널 오픈
	•	가격 스케일 바: 0원부터 최고가까지의 스케일 바에 4개 상품이 점으로 표시되어 가격 구조를 한눈에 파악
	•	고객 여정 플로우: 미끼 상품부터 최고가 상품까지 고객이 이동하는 흐름을 화살표로 시각화
웹앱 추가 가치:

	•	수익 시뮬레이션: '미끼 상품으로 100명이 유입되고, 전환율 10%로 계산하면...' 같은 간단한 수익 예측

활동 1: 나의 첫 팬 10명 만들기

	•	팬 전환 파이프라인: 후보 → 연락 → 가치 제공 → 반응 확인 → 팬 확보의 단계별 파이프라인 보드

	•	가치 제공 아이디어 추천: PART 1에서 파악한 기술/자원과 팬의 문제를 매칭하여 제공할 가치 추천

실물 플래너 형태: 핵심 활동 정의(표: 활동/시간/공간/규칙) → 방해 요소 제거(표) → 보상 시스템(일일/주간/월간) → 실행 회고.

	•	방해 요소 차단기: 좌(방해 요소) → 우(차단 계획) 연결형 입력. 방해 요소를 입력하면 AI가 차단 계획 예시 제안
	•	보상 설계: 일일/주간/월간 보상을 카드로 설정. 보상 달성 시 축하 애니메이션

	•	신호등 색상의 직관적 UI: 각 열이 실제 신호등 색상으로 배경 처리
	•	행동 계획 패널: 분류 후 각 영역별 행동 계획 입력. 특히 노란불 활동의 '개선 방법'을 구체적으로 기록

활동 6: 지속 가능성 점검 리스트
실물 플래너 형태: 재정적 건강(4항목)/정신적 건강(4항목)/사회적 건강(4항목) 각각 예/아니오/개선 필요 체크. 개선 계획 서술. 총평.
웹앱 UI 설계:

	•	건강도 게이지: 3개 영역의 건강도를 각각 게이지 바로 시각화 (예: 재정 건강 75%, 정신 건강 50%, 사회적 건강 100%)
	•	개선 필요 항목 하이라이트: '개선 필요'로 체크한 항목만 모아서 개선 계획 작성 화면으로 전환
웹앱 추가 가치:

	•	AI 우선순위 추천: '현재 가장 시급한 영역은 정신적 건강입니다. 먼저 충분한 휴식 시간을 확보하는 것부터 시작해보세요'
	•	정기 점검 알림: 매월 1회 지속 가능성 재점검 알림. 이전 점수와 비교하여 개선 추이 그래프

3. AI 코칭 시스템 설계
3-1. AI 코칭의 역할 정의
드림 플래너의 AI 코치는 사이먼 스큅의 철학과 방법론을 내면화한 가상의 코치입니다. 답을 직접 주지 않고 질문을 통해 사용자가 스스로 발견하도록 돕습니다.
역할
설명
안내자 (Guide)
각 활동의 목적과 방법을 자연스럽게 설명. 실물 플래너의 '작성 방법' 가이드를 대화형으로 전달
질문자 (Questioner)
사용자가 작성하는 과정에서 깊이를 더하는 추가 질문. 예: '혹시 이 기술을 배우게 된 계기가 있나요?'
분석가 (Analyst)
작성된 내용을 분석하여 패턴, 인사이트, 다른 활동과의 연결점 제시
격려자 (Cheerleader)
사이먼 스큅 스타일의 따뜻하지만 솔직한 격려. 진행 상황에 맞춘 동기부여
연결자 (Connector)
파트 간 데이터를 연결. 'PART 1에서 발견한 당신의 강점이 PART 3에서 이렇게 활용될 수 있습니다'

3-2. AI 코칭 톤앤매너
	•	따뜻하지만 솔직함: 과도한 칭찬이나 빈말은 하지 않음. '좋은 시작입니다'가 아닌 '재미있는 관점이네요. 여기서 한 단계 더 들어가볼까요?'
	•	행동을 촉구함: '생각해보세요'가 아닌 '지금 바로 3개만 적어보세요'. 실행 우선주의 철학 반영
	•	실용적인 톤: 추상적 조언 대신 구체적 예시와 행동 가이드
	•	사용자 수준에 맞춤: 초보자에게는 더 자세한 안내, 경험자에게는 핵심만 짚는 등 적응형 코칭
	•	한국어 자연스러운 존댓말: '~하세요', '~해보시겠어요?' 같은 부드러운 존댓말 사용

3-4. AI 데이터 활용 체계
AI 코치는 사용자가 이전 활동에서 입력한 데이터를 종합적으로 활용합니다. 이를 통해 일관성 있고 개인화된 코칭이 가능합니다.
데이터 흐름
활용 방식
PART 1 → PART 3
PART 1의 기술 목록, 자원 점수, 가용 시간, 절약 가능 금액이 PART 3의 MVP 구상 화면에서 참고 패널로 자동 표시
PART 2 → PART 3
PART 2의 Why 정의, 최종 아이디어, 시장성 분석 결과가 PART 3의 한 줄 제안 및 가설 설정에 자동 참조
PART 2 → PART 4
PART 2의 강점/약점이 PART 4의 Dream 5 네트워킹 전략에 반영. '당신의 약점인 ○○을 보완해줄 수 있는 동료를 찾아보세요'
전체 → 종합 리포트
PART 1~4의 모든 데이터를 AI가 종합 분석하여 '나의 여정 요약' 리포트 생성

4. 실물 플래너 + 웹앱 연동 설계
4-1. 실물 → 웹앱 (사진 촬영 → OCR 변환)
	•	실물 플래너에 손으로 쓴 내용을 스마트폰 카메라로 촬영
	•	웹앱의 '실물 플래너 가져오기' 버튼을 통해 사진 업로드
	•	AI OCR(광학 문자 인식)이 손글씨를 텍스트로 자동 변환
	•	변환된 텍스트를 사용자가 확인하고 수정한 후 해당 활동에 저장
	•	마인드맵, 표 등 구조화된 내용은 AI가 형태를 인식하여 웹앱의 해당 UI 요소에 맞게 배치

4-2. 웹앱 → 실물 (PDF 내보내기)
	•	웹앱에서 작성한 내용을 실물 플래너 페이지 형태의 PDF로 내보내기
	•	프린트하여 실물 플래너의 빈 페이지에 붙이거나 함께 보관 가능
	•	방사형 차트, 마인드맵 등 시각화된 결과물도 인쇄 가능한 형태로 변환

4-3. QR 코드 연동
	•	실물 플래너의 각 활동 페이지에 QR 코드를 삽입
	•	스마트폰으로 QR 코드를 스캔하면 웹앱의 해당 활동 화면으로 바로 이동
	•	실물로 작성하다가 AI 코칭이 필요할 때 즉시 웹앱으로 전환 가능

4-4. 동기화 대시보드
	•	실물과 웹앱 각각에서 작성한 활동의 완료 상태를 한 화면에서 통합 관리
	•	실물 플래너에서 완료한 활동은 수동으로 '실물에서 완료' 체크 가능
	•	전체 진행도는 실물 + 웹앱 모두를 합산하여 표시

기능
상세 설명
여정 지도 (Journey Map)

각 PART 완료 시 축하 팝업 + 특별 AI 메시지. 특정 활동 완료 시 미니 보상 (예: MVP 완성 시 '실험가' 칭호 획득). 10개/15개/20개 활동 완료 시 단계별 축하

8. 기술 스택 추천

10. 드림 허브 생태계 연동 설계
드림 플래너가 드림 허브의 다른 서비스들과 어떻게 데이터를 주고받는지 설계합니다. 이 연동을 통해 플래너는 독립된 앱이 아닌, 드림 허브 생태계의 핵심 레이어로 기능합니다.
10-1. 드림 브레인 → 드림 플래너
데이터 흐름: 드림 브레인에서 수집/분류된 생각과 아이디어가 플래너의 각 활동에 참고 자료로 제공
	•	브레인에서 자주 기록한 관심사/키워드가 PART 2 경험 마인드맵에 '추천 가지'로 자동 제안
	•	브레인에서 추출된 할 일 목록이 PART 3 MVP 구상의 실행 계획에 참조
	•	브레인의 3D 뇌 시각화에서 특히 밀도가 높은 주제 영역이 PART 2의 Why 발견에 힌트로 제공
구현 방식: 드림 브레인 API에서 사용자의 키워드 빈도, 주제 분류, 할 일 목록을 가져와 플래너 UI의 참고 패널에 표시

10-2. 다이얼로그 오브 드림스 → 드림 플래너
데이터 흐름: 다이얼로그 체험에서 나온 AI 분석 리포트('나와의 대화 리포트')가 플래너에 인사이트로 연결
	•	리포트에서 발견된 핵심 가치관, 열정 분야가 PART 2의 'Why 정의'에 참고 자료로 제공
	•	리포트의 감정/고민 분석이 PART 1의 '현재 상태 정의'에 깊이를 더함
구현 방식: 다이얼로그 리포트 데이터를 플래너가 읽어와 해당 활동 화면에서 '다이얼로그에서 발견한 것' 섹션으로 표시

10-3. 드림 플래너 → 드림 플레이스
데이터 흐름: 플래너에서 구체화한 정보가 플레이스의 프로필과 매칭에 활용
	•	PART 1에서 파악한 기술/역량이 드림 플레이스 프로필의 '보유 역량'에 자동 반영
	•	PART 2에서 발견한 꿈/Why가 플레이스의 '나의 꿈' 프로필에 연결
	•	PART 3에서 구체화한 아이디어/프로젝트가 플레이스의 '프로젝트 생성'으로 직접 이어짐
	•	PART 4의 '첫 팬 10명'과 'Dream 5 네트워킹'이 플레이스의 글로벌 매칭과 직접 연결
	•	플래너 완주 시 '플래너 완주' 배지가 플레이스 프로필에 자동 부여
구현 방식: 플래너 데이터를 플레이스의 프로필 시스템과 매칭 엔진에 API로 전달. 사용자 동의 후 자동 반영

10-4. 드림 플래너 → 드림 스토어
데이터 흐름: 플래너에서 만든 제품/서비스를 스토어에 올리는 연결
	•	PART 3에서 만든 MVP/제품을 '드림 스토어에 올리기' 버튼으로 직접 연결
	•	스토어의 상품 상세 페이지에 '이 제품은 드림 플래너에서 시작되었습니다' 스토리가 자동 생성 (사용자 동의 시)
	•	플래너의 가치 사다리(미끼→저가→중가→고가)가 스토어의 상품 구성에 참조

10-5. 드림 플래너 → 드림 카페
데이터 흐름: 오프라인 경험과 온라인 플래너의 연결
	•	드림 카페에서 '드림 플래너 워크숍' 프로그램 운영. 참가자들이 함께 특정 활동을 수행
	•	카페에서 만난 사람과 플래너의 특정 활동(예: Dream 5 네트워킹)을 공동으로 작업하는 기능
	•	카페 내 다이얼로그 오브 드림스 체험 후 플래너로 자연스럽게 연결되는 동선 설계

생태계 연동 전체 흐름도
드림 브레인 (기록) → 다이얼로그 (발견) → 드림 플래너 (계획) → 드림 스토어 (판매) + 드림 카페 (교류) + 드림 플레이스 (연결)
플래너는 이 흐름의 중심에서 '계획을 구체화하는' 핵심 허브 역할을 합니다. 위(브레인, 다이얼로그)에서 데이터를 받아오고, 아래(스토어, 카페, 플레이스)로 데이터를 내보내는 양방향 연결 구조입니다.
